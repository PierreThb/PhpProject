\AppBundle

\AppBundleEntity

Classes

Item Item is related to an agenda, has a name, a proposer and a number
ItemAction ItemAction is the entity for each action which can be performed on an item and is defined by the related item, the state of the action, a description, an implementer and a deadline.
Meeting A Meeting is related to a Project. Each Project can have several Meeting.
MeetingAgenda Each meeting has a MeetingAgenda.
MeetingAttendance Each meeting has a MeetingAttendance for each member of the team.
MeetingMinutes Each meeting, when his deadline is crossed, has a MeetingMinutes which is defined by the related meeting, a set of Presence, a set of comments and a set of minuteItem.
MinuteItem Each MeetingMinute contains at least three MinuteItem which are defined by an item, the related MeetingMinute, an action, a comment and a postponed boolean.
MinutesComment Each member present at the meeting can add MinutesComment. It is defined by the user who added the comment, the related MeetingMinute and the content of the comment.
Presence Each member of the team has a Presence object for each meeting. It is defined by the related user, the related MeetingMinute and the string type which describe the Presence.
Project The admin can create Project. Each project is defined by a name, a leader, a secretary, a set of users, a boolean to lock and unlock the project and a set of meetings (is null when the project is created)
User The admin can create User. Each user is defined by a name, has a password (crypt) and a boolean to be or not an admin user.
UserRequest Each member make request to the meeting leader about the MeetingAgenda. Each UserRequest is defined by teh related MeetingAgenda, the user who did the request, the related item, the type of request, the state of the request and the content.
